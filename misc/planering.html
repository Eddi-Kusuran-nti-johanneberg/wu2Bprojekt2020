4. Testning

Testning på dator

På min dator ser allt mycket bra ut. Sidan är bra anpassad till mobil och dator och ser bra ut på båda. Sidan är också anpassad efter olika webbläsare

Allting skalar sig självt på ett bra sätt och visuellt ser den anpassad ut på ett sätt som passar mobil respektive dator.

Testning på mobil

Detta testet utförde jag på min iPhone Xs i Safari. Testet gick överlag bra sidan såg bra ut och fungerade korrekt.

Det negativa med testet var att jag inte hade tänkt på barena som safari har längst upp och ned på skärmen. Detta gjorde så att jag var tvungen att skrolla för att se toppen och botten av sidan.

Detta ska jag tänka på nästa gång jag designar något för mobil.

HTML Validering

Testet på min index sida gav mig några varningar och några error. De två error jag fick handlade om att en bild som jag hade på sidan hade mellanslag i sitt namn, detta ändrade jag på väldigt enkelt.

Två varningar klagade på att jag använde elementet h1 i en text som inte var längst upp på ett annat elemt. Jag ändrade den till h2 och alla headings under ändrade jag också på ett steg nedåt.

De två sista varningarna tyckte jag inte var något jag behövde bry mig om då de klagade på att jag inte hade ett heading element i en section. Jag vill inte ha någon heading i den sectionen så jag ignorerar denna varning.

På min andra html sida fix jag ett error och en varning som handlade om samma problem, jag hade använt id två gånger. Jag ändrade snabbt det till class och ändrade i mitt css dokument (från # till .).

På min tredje och fjärde html sida fick jag inga error eller varningar.

CSS Validering

På valideringen på min css fick jag inga error eller varningar.

Det första jag skulle vela lyfta fram är användandet av Github. Även om jag nu tycket att Github är väldigt bra så tror jag att läraren skulle kunna ha introducerat det till eleverna på ett bättre sätt.

Jag själv tyckte det var lite flummigt angående uppstarten av programmet och fick prova mig fram vid vissa tillfällen för att få det att funka.

Att ha dokumentation på detta sätt (att skriva i ett html dokument) tycket jag är jätte bra. Det får oss att bli mycket mer bekväma och vana vid att använda både hmtl och visual studio code.

Läraren har varit vädligt tydlig med kunskapskrav och vad som krävs av eleverna, detta är bra.

Angående själva arbetsprocessen så skulle jag säga att det har gått bra men med vissa negativa svängar här och där.

Vid vissa tillfällen så har fastnade jag helt och hade verkligen inte någon som helst aning om hur jag skulle gå till väga för att lösa problemet jag hade.

Som tur är så hade jag alltid kamrater att fråga om hjälp samt så fanns det mycket information på nätet som jag kunde använda mig av, speciellt W3Schools.

En sak som jag tänker på och som jag kommer att ha i åtanke vid nästa projekt är att anpassa mobil sidan till Safari. Som jag skrev lite längre upp så gjorde barena på Safari att man var tvungen att skrolla upp och ned vid vissa tillfällen.

Detta kan jag förbättra genom att göra sidan lite kortare på längden till nästa gång.

Ett problem som jag hade i processen var att jag sköt upp mycket även om jag hade enormt mycket tid att t.ex jobba hemma.

Det fanns såklart andra faktorer här så som annat skolarbete men i webbutvecklingen så måste jag effektivisera mitt arbete och inte skjuta upp det.

Om jag hade gjort så från början så skulle jag ha kunnat spara tid då jag inte skulle skriva samma saker flera gånger. Det här tar jag med mig tll nästa projekt.
